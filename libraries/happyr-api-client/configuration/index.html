Here is a list of the configuration parameters. Take a look in Configuration.php if you want to change any of these values.

Mandatory parameters

username

The username for happyrecruiting.se. You will get this value when you are authorised to use the API.

token

This is the passphrase that to got from happy recruiting.se It is not the same as your password. You will get this value when you are authorised to use the API.

Optional parameters

baseUrl

This is the url to the API. The default value is "http://api.happyr.com/api/". The baseUrl must always end with a slash (/).

debug

If you want to debug the API, sent this boolean to true. You will get some debug prints and we wont hide errors.

enableExceptions

This boolean enables you to get exceptions from the API. With the value set to false we will try to catch all exceptions and deliver expected/acceptable result. The default value is false.

format

What format should it be on transmitted data. The formats that are supported is JSON and XML. The default value is XML because it has less change of breaking when the API is updated.

httpRequestClass

This parameter tells the client what class should be used to make a HTTP request. The default is \HappyR\ApiClient\Http\Request\CurlRequest but you may enter any class you want. The class must implement the \HappyR\ApiClient\Http\Request\RequestInterface.

serializerClass

What serializer class do you want to use. The default is \HappyR\ApiClient\Serializer\JmsSerializer but you may change it if you like The class must implement the \HappyR\ApiClient\Serializer\SerializerInterface.
